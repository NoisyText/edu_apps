XML Config Parameters

Introduction

There's a pretty slick option in our configuration interface that lets you use variables that are substituted with user-entered values in your generated XML configuration. For example, if your launch URL was http://my.app/launch?id=<customer_id>, it would be possible to ask the user as part of the directory interface for a value for customer_id.

The redirect app is a good example that uses some config parameters to specify the link name and redirect URL.

There are two steps to make variable substitution work. The first is to use our variable notation in form entries. Variables are entered as letters and underscores surrounded by double curly-braces. So to make the previous example work, you would put http://my.app/launch?id={{ custom_id }} as the field value.

That's step one. Step two is to add a field in the "XML Config Parameters" section. Name should be the variable name, in this case custom_id. Prompt is the text users will see next to a form element. Req'd is whether the value is required, Type lets you pick text field or checkbox, and Value is the default value.

Side note: Once you add at least one parameter, you'll see two new fields, Variable Name and Variable Description. These fields can be ignored, or you can use them to define a name and description that will be based on variables that will only show up in the XML configuration. These fields won't affect the app name and description as it appears in the directory.

Here's a list of all the fields that support variable substitution:

  • LTI Launch URL
  • Custom Field Values
  • Variable Name
  • Variable Description
  • Multiple Config URLs
  • Course Nav Link Text
  • Account Nav Link Text
  • User Nav Link Text

Extra Goodies

There are some hard-coded parameter names that are used by the "Help Me Build My Configuration" option to make config easier. Here's the list, and how each of them affect the generated XML:

Parameter Description
course_nav If the "course navigation" extension is selected, then this parameter can be used to let the user decide whether the app shows up in course navigation or not. We recommend picking "Checkbox" as Type with a Value of "1".
user_nav If the "user navigation" extension is selected, then this parameter can be used to let the user decide whether the app shows up in user navigation or not. We recommend picking "Checkbox" as Type with a Value of "1".
account_nav If the "account navigation" extension is selected, then this parameter can be used to let the user decide whether the app shows up in account navigation or not. We recommend picking "Checkbox" as Type with a Value of "1".
editor_button If the "editor button" extension is selected, then this parameter can be used to let the user decide whether the app shows up in the rich editor or not. We recommend picking "Checkbox" as Type with a Value of "1".
resource_selection If the "resource_selection" extension is selected, then this parameter can be used to let the user decide whether the app shows up in module link selection tool or not. We recommend picking "Checkbox" as Type with a Value of "1".
